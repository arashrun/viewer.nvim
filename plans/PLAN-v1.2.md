# viewer.nvim v1.2 计划

## 目标

- 在现有 markdown 预览能力之外，新增离线 API 文档渲染与查询能力。
- 让 `viewer.nvim` 能和 `nview` 协作浏览本地文档库，而不是只渲染当前编辑缓冲区。
- 优先兼容 Zeal / Dash 可用的 `docset` 格式，保证能复用现有离线文档生态。
- 保持当前 TCP JSONL 架构不变，尽量在现有 `viewer.nvim` / `nview` 代码路径上扩展。

## 已确认范围

1. 支持加载和浏览离线 API 文档。
2. 支持从 `nvim` 发起交互查询，例如关键词搜索、条目跳转、结果预览。
3. 支持 `docset` 数据源，至少覆盖常见的 Zeal / Dash 兼容结构。
4. 继续保留现有 markdown 预览，不破坏 v1.0 / v1.1 的能力。

## 设计约束

- 先做本地离线文档，不依赖网络检索。
- 以“查询 + 预览 + 跳转”为核心，不先做复杂的在线同步或账号体系。
- `docset` 兼容优先保证读取与索引，复杂扩展协议后置。
- `viewer.nvim` 仍然只负责发起查询和接收结果，主要渲染与窗口交互继续在 `nview` 侧完成。
- 查询面板和预览面板是同一个窗口,nview同一时间只能显示其中一种
- 本地离线文档这里的本地是指在nview运行所在的主机
- 文档数据目录和文档包根目录由 `nview` 本地决定，`nvim` 只传查询意图，不传具体文档路径。

## 数据目录

- 默认文档根目录建议使用用户数据目录下的 `viewer.nvim/docs`。
- 文档包保持自包含结构，不拆散 `docset` 内部目录。
- 默认缓存目录建议放在文档根目录下的 `cache` 子目录，或者单独配置为 `viewer.nvim/docs-cache`。
- 目录职责分层如下：
  - `docs_root`：文档库总入口
  - `docset` 子目录：单个离线文档包
  - `cache`：索引缓存、派生元数据、查询结果缓存
- 后续实现优先支持单根目录，结构稳定后再扩展为多根目录列表。

## 技术方案

### 1. 文档数据模型

- 抽象出统一的文档条目模型，包含：
  - `name`
  - `path`
  - `fragment` / `anchor`
  - `kind`
  - `source`
  - `score`
- 对不同来源的数据先统一到同一个结果结构，再交给前端展示。

### 2. docset 兼容

- 支持识别 docset 根目录。
- 读取 `Contents/Info.plist` 或等价元数据，确认 docset 名称、版本和入口页。
- 支持扫描 `docSet.dsidx` 中的条目索引。
- 将 docset 条目映射成统一查询结果。
- 文档正文读取先从本地 HTML 文件直接渲染。

### 3. 查询协议

- 在 `lua/viewer/protocol.lua` 扩展新的消息类型，用于：
  - 查询关键字
  - 返回搜索结果
  - 打开某个结果
  - 请求结果预览
- `nvim` 侧提供命令和可选快捷键入口。
- `nview` 侧增加查询面板或结果列表视图。

### 4. 渲染与交互

- 文档预览先沿用 `nview` 的 HTML 渲染窗口。
- 搜索结果页支持列表浏览和键盘选择。
- 打开结果时，支持定位到页面锚点或具体标题位置。

## 里程碑

1. 统一文档模型与协议草案完成
2. `docset` 读取与索引能力完成
3. `nvim` 查询命令完成
4. `nview` 查询面板与结果展示完成
5. 跳转与预览联动完成
6. Windows 产物编译并验证

## 验收条件

- 能加载本地 `docset` 并完成关键词搜索。
- 能从 `nvim` 触发查询并在 `nview` 中看到结果。
- 能打开某个条目并显示对应文档内容。
- 现有 markdown 预览能力保持可用。
- Windows 下可以成功编译生成 `./bin/nview.exe`。

## 风险点

- `docset` 的元数据和索引格式存在版本差异，需要先定义最小支持集合。
- 文档正文可能是 HTML、Markdown 或混合内容，需要统一渲染边界。
- 查询交互需要兼顾键盘操作和窗口焦点切换，避免影响现有预览体验。
- 如果结果面板和预览面板同时存在，需要明确布局与滚动同步策略。
